## 一、知识库结构问题

### 1.1 冗余的"Untitled document"节点
- **现象**：知识库目录中出现大量"Untitled document"（未命名文档），包括原始的"首页"节点和嵌套的空白文档
- **根因**：创建知识空间时飞书自动生成了"首页"节点；`create_wiki_node` 未设置 `node_title` 或节点创建后未被正确命名
- **影响**：知识库目录混乱，用户不知道这些空白文档是什么
- **修复**：
  - 删除所有"Untitled document"节点（飞书API不支持删除节点，需手动在客户端操作，或后续研究`/delete`端点）
  - 确认 `create_wiki_node` 的 `node_title` 参数正确传递（当前代码已传，但可能API在某些情况下未生效）

### 1.2 多维表格应以嵌入方式创建，而非独立创建后迁移
- **现象**：先创建了4张独立多维表格，再用 `move_docs_to_wiki` 嵌入到知识库。嵌入后原表格仍存在
- **根因**：`TableService._create_year_table()` 直接调用 `create_bitable`，创建了应用自有空间下的表格，然后才用 `embed_tables.py` 迁移
- **改进**：应直接在知识库节点下创建多维表格节点（`obj_type: "bitable"`），避免两步操作产生冗余

### 1.3 用户期望：知识库只需多维表格，不需要独立的课程文档节点
- **现象**：铭培希望知识库只展示多维表格，课程文档通过表格中的"学习指南"链接访问
- **当前状态**：知识库同时有多维表格节点和18个课程文档节点（树状结构），显得冗余
- **改进**：去掉 `WikiBuilder.build_course_nodes()` 步骤，课程文档直接作为独立文档存在（不在知识库树中显示），通过表格链接跳转

## 二、多维表格内容问题

### 2.1 表格多出空行
- **现象**：大一课程表有约10行空行，实际数据只有6行
- **根因**：`create_bitable` 创建时飞书默认会预生成一些空行
- **影响**：表格看起来不整洁
- **修复**：部署后清理空行，或在填充数据后删除多余记录

### 2.2 冗余的默认字段
- **现象**：表格中存在"A=文本"、"单选"、"日期"、"附件"等飞书默认创建的字段
- **根因**：`create_bitable` API 创建表格时自带默认字段（课程名称等），`TableService._create_year_table()` 又用 `BITABLE_COURSE_FIELDS` 添加了自定义字段
- **改进**：创建表格后立即删除默认的冗余字段，只保留我们定义的9个字段

### 2.3 URL字段格式
- **现象**：直接传字符串URL到URL类型字段会报 `URLFieldConvFail`
- **根因**：飞书Bitable的URL字段需要 `{text, link}` 格式
- **状态**：✅ 已在 `link_service.py` 中修复

## 三、文档生成问题

### 3.1 文档内容写入了新创建的文档，而非知识库节点文档
- **现象**：`doc_generator.py` 先 `create_document` 创建新文档再写入内容，但学习指南链接指向的是知识库节点文档（空的）
- **根因**：设计缺陷——应该直接往知识库节点的 `obj_token` 对应的文档里写内容
- **状态**：✅ 已修复，改为优先使用 `wiki_builder.get_course_node()` 获取 `obj_token`

### 3.2 LLM返回内容偶尔解析失败
- **现象**：智谱GLM-4-flash偶尔返回非标准JSON（如被截断），触发"LLM返回内容无法解析为JSON"
- **影响**：部分课程文档使用默认占位内容
- **改进**：
  - 增加重试机制（解析失败时重新调用LLM）
  - 使用更强的JSON解析（容错处理截断的JSON）
  - 考虑升级到GLM-4模型提高输出稳定性

## 四、权限与环境问题

### 4.1 飞书知识空间权限需通过群组间接授权
- **现象**：直接将应用添加为知识空间协作者不够，需要先创建群、将应用加为群机器人、再将群添加为空间成员
- **状态**：✅ 已解决，已记录到用户配置指南

### 4.2 docx block API端点变化
- **现象**：`/children/batch_create` 返回404，正确端点是 `/children`
- **状态**：✅ 已在 `feishu_service.py` 中修复

### 4.3 创建知识库节点必须带 `node_type: "origin"`
- **现象**：不传 `node_type` 会报 `field validation failed`
- **状态**：✅ 已在 `feishu_service.py` 中修复

2. 创建学年节点（仅目录） → ✅（保留）
3. 在每个学年节点下创建多维表格（`obj_type: "bitable"`） → 待实现

5. 填充课程记录 → ✅

7. 关联学习指南链接 → ✅
