## 设计决策

1. **为什么将 auto/ 移到 log/auto/**：auto/ 下的文件是自动生成的报告，本质上是日志输出，与 log/ 下的手动日志同属一类。

2. **为什么不重组 services/ 内部**：当前 services/ 下的各文件已经职责单一（每个文件一个服务类），代码质量良好，无需拆分。

3. **为什么保留 services 中的 `sys.path.insert`**：services 内部使用相对导入（`from config import ...`），因为它们是被 deploy.py 以 `from ppe_demo.services.xxx` 的方式导入的，此时 Python 的包上下文已正确建立，`sys.path.insert` 作为兼容措施保留不会造成问题。

4. **数据文件是否应该 gitignore**：materials.json 等包含本地绝对路径（如 `D:\c盘转移\...`），不适合提交到公开仓库。已在 .gitignore 中排除，首次使用时需从 `.env.example` 模板或文档中获取格式参考。
