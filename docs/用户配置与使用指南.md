# PPE云端智能大礼包 - 用户配置与使用指南

> 本文档面向想要部署PPE云端智能大礼包的同学。按照以下步骤完成配置后，系统会自动构建知识库结构并生成课程文档。

---

## 一、前提条件

- 一个飞书账号（南开大学邮箱即可注册）
- 飞书开放平台已创建自建应用
- Python 3.8+ 环境
- 本项目代码已克隆到本地

---

## 二、飞书应用配置

### 2.1 创建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)，使用飞书账号登录
2. 点击「创建应用」→「自建应用」
3. 填写应用名称（如"PPE大礼包"），选择组织

### 2.2 获取凭证

1. 进入应用 →「凭证与基础信息」
2. 记录 **App ID** 和 **App Secret**

### 2.3 开通权限

进入「权限管理」，搜索并开通以下权限（全部选择**应用身份**）：

| 权限名称 | scope | 用途 |
|----------|-------|------|
| 查看、编辑和管理知识库 | `wiki:wiki` | 在知识库中创建节点 |
| 云文档 | `docx:document` | 创建课程云文档 |
| 查看、评论、编辑和管理多维表格 | `bitable:app` | 创建课程表格 |

⚠️ **重要**：权限开通后，点击「创建版本并发布」，否则权限不会立即生效。

### 2.4 手动创建知识空间

由于飞书平台限制，**创建知识空间不支持应用身份调用API**，需要手动完成：

1. 打开飞书 →「知识库」
2. 点击「新建知识库」，名称填写 **PPE云端智能大礼包**
3. 创建完成后，点击知识库设置，在浏览器地址栏中找到 `space_id`
   - 地址格式：`https://xxx.feishu.cn/wiki/space/这里是space_id`
   - 复制这串数字/字母

### 2.5 将应用添加为知识空间成员

1. 打开刚创建的知识空间
2. 点击「空间设置」→「成员管理」
3. 点击「添加成员」，搜索你的应用名称并添加
4. 设置权限为「可编辑」或「管理员」

这一步是必要的，否则应用无法在知识空间下创建节点。

## 三、本地配置

### 3.1 安装依赖

```bash
cd PPE-CloudSmart-GiftBox
pip install -r requirements.txt
```

### 3.2 配置环境变量

复制 `.env.example` 为 `.env`，填写以下信息：

```env
FEISHU_APP_ID=你的App ID
FEISHU_APP_SECRET=你的App Secret
ZHIPU_API_KEY=你的智谱API Key（可选，用于AI生成文档内容）
```

### 3.3 配置知识空间ID

编辑 `ppe_demo/config.py`，将 `SPACE_ID` 设置为你在2.4步中获取的知识空间ID：

```python
SPACE_ID = "你在浏览器地址栏中看到的space_id"
```

### 3.4 配置课程列表

编辑 `ppe_demo/config.py` 中的 `COURSES_BY_YEAR`，按你的专业课程填写。每门课需要：

```python
{
    "name": "课程名称",
    "teacher": "授课老师",
    "semester": "大一上",  # 开课学期
    "type": "必修",        # 必修/选修
    "exam": "闭卷"         # 考试形式
}
```

## 四、运行部署

### 4.1 分步部署（推荐首次使用）

```bash
# 步骤1：在知识空间中创建学年节点和课程节点
python deploy.py --mode wiki

# 步骤2：创建多维表格并填充课程记录
python deploy.py --mode tables

# 步骤3：生成课程云文档（AI生成内容）
python deploy.py --mode docs

# 步骤4：将文档链接关联到多维表格
python deploy.py --mode link
```

### 4.2 一键部署

```bash
python deploy.py --mode full
```

## 五、使用说明

### 5.1 用户体验

部署完成后，同学们的使用路径：

1. 打开飞书知识库
2. 进入"PPE云端智能大礼包"知识空间
3. 选择对应学年
4. 打开课程多维表格，浏览所有课程
5. 点击某门课程的「学习指南」链接
6. 在课程文档中查看概述、难点、资料等

### 5.2 维护与更新

- **添加新课程**：修改 `config.py` 中的 `COURSES_BY_YEAR`，重新运行 `--mode tables`
- **更新资料**：修改 `data/materials.json`，重新运行 `--mode docs`
- **添加心得体会**：修改 `data/experiences.json`，重新运行 `--mode docs`

## 六、常见问题

### Q: 权限已开通但还是报错怎么办？

权限开通后需要「创建版本并发布」才会生效。如果已经发布，等2-3分钟让token刷新。

### Q: 创建知识空间报 99991663 错误？

这是正常的。创建知识空间需要用户身份（user_access_token），不支持应用身份。请手动在飞书中创建，见步骤 2.4。

### Q: 应用无法在知识空间中创建节点？

请确认已将应用添加为知识空间的成员（步骤 2.5），并赋予编辑权限。

### Q: 文档内容质量不好？

文档内容由AI（智谱GLM）基于心得体会生成。心得体会越详细，文档质量越高。可以在 `data/experiences.json` 中补充更多高分学长学姐的心得。
